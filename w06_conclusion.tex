\chapter{总结与展望}\label{ch:conclusion}

在前面各章的论述中，作者介绍了聚合重构分布式终端系统与承载它的底层网络~――~移动个人网络及其若干关键机制，通过这一自组织系统，用户周边的智能设备得以协同工作，并为用户提供连续、一致、最佳的业务体验。

各种异构网络的融合是通信网络技术发展的方向，个人周边智能化是信息技术发展的必然趋势，在~B3G~乃至更远的将来，业务将不仅仅是已有业务的简单集成和数量上的增加，网络智能将与泛在的个人周边智能协同工作，利用无所不在的智能感知、计算、存储、通信、执行设备，主动适应周边环境的变化，依照用户的个人喜好，始终为用户提供连续、一致、最佳的业务体验。

然而，尽管移动泛在的融合已然被认为是大势所趋，业务的演进面临着很多技术上的挑战。目前关于个人周边设备协同工作的研究一直没有考虑移动性的问题，当前的广域网络的移动性管理机制渗透到个人环境中的众多智能周边设备面临着能力和效率的双重挑战，而已有的智能设备互操作机制则并未涉及到移动性，这使得这些泛在与移动的结合仍然无法平滑自然地实现。

在本文中，我们将个人周边的智能设备协同为一个时刻跟随用户的动态聚合重构的分布式终端系统，一方面可以利用不同接入设备和接入网络，克服异构网络对通信服务的影响，更可以协同发挥各个设备的服务能力，提供更好的业务体验，当然，这也面临着一些技术挑战：
\begin{itemize}
	\item 如何对分布式终端系统的系统架构、行为特征进行建模；
	\item 如何帮助应用可以及时发现新的业务资源并利用它们；
	\item 如何适应用户移动过程中，系统成员大量更迭对正在进行的业务和系统本身的冲击。
\end{itemize}

基于我们在此领域进行的前期架构研究，论文中描述了聚合重构分布式终端的应用场景和基本概念，归纳了终端的工作状态和系统架构，阐述了作为分布式终端组成元素的各个设备的结构和抽象接口，并提出了分布式终端系统的有中心和无中心两类组织方式。

我们注意到，分布式终端所在的自组织的网络结构并非一个普通的移动自组织网络，而是紧密围绕着~``用户''~这个中心，面向业务的个人网络，因此，我们称之为~``移动个人网络''，并对移动个人网络的连通性和移动性特征，进行了归纳和分析，总结出短距离性、异构性和组移动性三大特征，并在前人所研究的移动自组织网络的移动性模型的基础之上，结合移动个人网络的特定情况，建立了移动个人网络的组移动性模型。

基于移动个人网络的连接和移动性特征，我们给出了基于组移动性的移动个人网络的服务发现机制，在保障应用可以及时发现系统中可用资源的变化的情况下，有效降低应对大量设备同时加入或退出情况下服务发现信令对网络正常工作的冲击。

更进一步，我们还对服务发现机制和系统组织管理的底层支撑基础设施~――~分布式的数据存储机制进行了研究，通过比较有中心存储、非结构化无中心存储和结构化无中心存储机制，给出了适应于移动个人网络的行为特征的无中心分布式存储机制，具有良好的可靠性、稳定性和可伸缩性，与已有结构化无中心存储方法相比，占用更少的带宽资源，并且更能适应移动个人网络中节点的组移动性。

不过，移动泛在融合的萌芽才刚刚开始，对这个领域的研究仍然很不充分，虽然分布式终端系统与移动个人网络技术在将来可能会进入人们的日常生活，但是当前仍有许多不足和有待解决的问题，这里仅仅探讨一些在下一步将要或亟待解决的问题：
\begin{itemize}
	\item 安全性与私密性问题。终端是极为个人化的信息设备，当终端分布化，用户相关的信息在终端内部传输时，便史无前例地面临监听与破坏，除了~IP~层和应用层的加密和完整性检查之外，设备加入过程中的鉴权与授权过程同样需要考虑，结合设备的组移动性，可靠地成组鉴权授权必将可以提高鉴权授权的效率，但需要考虑确保安全性的问题。同时，当分布式终端中的设备的所有权不归属用户的时候，如何控制设备对敏感数据的访问权限，如何对安全性事件的进行监控与记录，都需要研究和规范化。
	\item 多用户问题。论文中的研究始终围绕单一用户周边的多个设备展开，而并未探讨多个用户的移动个人网络的范围发生交叠的情况，而这种情况不论是在理论上还是实际应用当中都非常有意义。当多个用户的控制范围交互的时候，如果多个用户同时希望使用同一设备，如何确定设备被哪个用户占用；反之，当系统中存在和其他用户共用的设备的时候，应用应如何选择所需要的服务；当多个用户同时存在时，如何避免相互之间的干扰和同时存在的安全性问题也需要考虑；另外，如果多用户共用设备提供的付费服务资源，会对运营计费产生如何影响。这些都是多用户情况所需要特别考虑的问题。
	\item 设备的移动性分组问题。目前对移动个人网络的研究之中，我们没有考虑移动设备的动态分组，尽管目前有很多自组织网络中基于移动性的分组方法，但是，移动个人网络独特的移动性行为、网络的异构性以及短距离特性都会使得已有的移动性分组算法并不能发挥较好效果，设备初始化的时候显式设定仍然是实际应用中最有效的手段，但是学习算法的加入可以降低设置的复杂度、增加一定的灵活性，因此也将在下一步被考虑。
\end{itemize}

总之，就我们目前的研究基础看来，有待进一步研究的内容仍然很多。首先，移动个人网络作为个人周边智能设备的协作体、分布式终端系统的载体，本身的部分问题仍然有待完善与解决。其次，
