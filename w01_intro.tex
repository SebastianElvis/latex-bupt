\chapter{绪论}\label{ch:intro}

``融合''~是信息技术在~21~世纪前~20~年的主旋律，从信息设备的通信、计算、消费电子的~``3C~融合''，到信息传输载体的电信、数据、广电~``三网融合''，都已经开始发生并深刻地影响着信息领域的各种技术的发展。移动通信技术从第三代之后，融合各种空中接口技术的接入网的承载能力不仅可以满足原有电信业务的需求，而且可以支撑各种数据业务和广电业务，从而促进了业务的融合; 而另一方面，通信终端则已经拥有了更强的处理能力并集成了多种外设技术，使之不仅是一个通信终端，更是继个人计算机~(PC)~之后的新一代个人信息处理中心。

在技术发展的推动之下，人们对于业务体验的需求也进一步被激发，从而对技术的进一步发展产生了新的需求。人们不能满足仅仅是原有业务的简单集成和数量上的增加，个人周边所暗藏着的各种物质、能量与信息的处理能力的\textbf{协作}已经初见端倪，并将使人们的生活发生一场新的变革，而这场变革在移动通信方面的表象就在于：协同利用用户个人周边的无线智能设备，为移动用户提供连续、一致、最佳的个人业务体验。这里，所谓``\textbf{连续}''是指当用户移动时，业务进行不被间断; 所谓``\textbf{一致}''，是指业务进行当中，当需要进行调整的时候将依照用户喜好来进行调整; ``\textbf{最佳}''则是指充分利用用户周边设备的能力来提供用户满意的业务。

\section{移动通信与泛在计算技术的融合}

毫无疑问，移动业务的不断发展和计算技术的进步有着密切的关系。移动计算技术、分布式系统技术的发展预见了所谓的``泛在计算''~ (Ubiquitous Computing, Pervasive Computing, 又称为无处不在的计算)~时代的到来，``泛在计算''~的核心特征在于让各种智能设备广泛地深入人们的周边，真正地方便人们的生活，而不是成为人们生活的主角。

泛在计算技术起源于分布式计算，自从计算机之间开始通过网络互联开始，资源的共享成为了计算领域的一个重要的研究点，网格计算技术更是通过共享普通~PC~上的剩余计算能力达到了超越超级计算机的计算能力。随着无线通信技术，特别是移动技术的发展，使得大量的计算设备移动化，当它们成为分布式系统的成员的时候，移动计算便成为了一门新兴的技术，系统中的资源变化更加频繁，移动计算系统需要被更精巧地设计，以避免移动性所带来的资源不可用，并进一步利用移动性所带来的更多可用资源。而更进一步，当微处理、微机电以及短距离通信技术的进步使得人类所生活的环境变成一个智能空间，

泛在计算这个~Mark Weiser~在~20~世纪~80~年代末~90~年代初就已经预言了的愿景正逐步接近实现。在未来，用户周边将存在更多的服务用户的智能设备，如各种终端、智能家 电、传感器设备、车载设备、可穿戴设备等，据~WWRF~预测，到~2017~年，全球将有~7~万亿的无线设备服务着~70~亿的人口，这些设备将不是孤立的，他们会互相影响、彼此支援乃至协同工作，这些功能各异的设备的协同工作会极大延伸用户的信息乃至物质与能量的处理能力，而它们作为未来业务的展现工具，也让我们相信，未来的移动业务会更加丰富。欧盟第六框架中的一个研究重大主题~``Ambient Intelligence''~即周边智能化，意图多方位多角度的探讨在信息基础设施在实现周边的智能化环境时可能遇到的种种问题。日本经过近十年的信 息社会的发展，于~2004~年确定~``Ubiquitous Japan''~的发展目标来取代之前的~``E-Japan''~的目标，开展系列研究计划构建未来泛在的信息社会。

在这样的背景之下，未来的业务需要组织、利用用户周边的智能设备来为用户提供连续、一致、最佳的业务体验。这就是所谓的移动泛在业务环境~(\textbf{MUSE}, Mobile Ubiquitous Service Environment)。

\section{业务的连续性}

连续性的业务服务体验是移动业务的初始需求，早期的移动业务便是直接赋予固定网络中的业务在移动过程中的连续性而得的，从用户体验的角度来看，如果不考虑无线传播对信号质量造成的影响，这些业务与它们的固定网络中的前身并没有太大区别，即使用户移动到了归属运营商的无线网络覆盖范围之外，由于业务的高度规范性，其业务提供的连续性和业务内容的一致性也有足够的保障。

当网络在固定业务移动化之后，人们逐渐开始进一步利用移动通信所特有的移动性和个人性以获取更多的便利与更丰富的业务体验，也同时为业务提供者带来更多的利益，这也就是所谓的增值业务。不过，与传统的电信业务不同，不同运营商或服务提供商所提供的增值业务内容各异，不同终端展现业务的能力也具有很强的差异性，业务形式受到不同承载网络、服务提供商、终端设备的影响也相比于传统业务更大，而用户个性化需求和市场驱动使得业务本身的规范化变得十分困难。

在这样的背景之下，虚拟归属环境~(VHE)~概念应运而生，它把移动性的需求推广到了增值业务，超越了简单、规范的基本业务的范畴。这里的移动性或称连续性实际上是一种可移植性或互操作性的概念，也就是说，变换了工作平台和某些业务上下文内容之后，业务仍然可以以接近用户需求的方式服务用户。这种可移植性所依赖的就是标准化，虽然多种多样的增值业务难于标准化，但我们可以退而规范构成业务的基本元素，这些元素必定是有限的，正所谓``声不过五，五声之变， 不可胜听也。色不过五，五色之变，不可胜观也''。

首先被抽象的是网络的各种业务控制功能，这项工作在智能网技术中就已经开始了，并且向着越来越开放的方向演进。在~IMS~之中，更为简单的~SIP~协议取代了~7~号信令完成业务控制的功能，而~Parlay/OSA~乃至~ParlayX~更通过~CORBA, Java, Web Service~等计算机软件开发人员耳熟能详的接口，把业务控制以标准接口的形式开放给了第三方的业务提供商，从而极大地提高了增值业务的可移动性。在业务内容、各种信息描述方面，近年来，诸如~RDF, CC/PP~等~XML~格式的日益普及以及对于~Profile~存储与交换的规范如~GUP~的制定也帮助业务的行为在很大程度上标准化了。而在终端方面，除了基于~SIM~卡的~STK~规范以外，包括~J2ME, Brew~等中间件技术和开放移动联盟~(OMA)~的~WAP~技术等，都在试图建立一套标准的终端业务套件，从而帮助业务实现跨越不同终端的移动性。

\section{移动泛在融合在个人域遇到的挑战}

然而，尽管移动泛在的融合已然被认为是大势所趋，业务的演进面临着很多技术上的挑战。 目前关于个人周边设备协同工作的研究一直没有考虑移动性的问题，而移动性造成的影响将对个人域网络和广域网络造成一定的冲击。

IP~技术是电信网络演进的方向，在~3GPP SAE~中，核心网也会向着减少层次、采纳~IP~技术的方向发展。虽然~IPv6~的自动配置技术、Mobile IPv6~技术以及网络移动性技术~(NEMO)~在一定程度上从网络连接层面解决了众多设备的移动性问题，但它们并未涉及到业务层面，设备之间组织以及它们的互操作所依赖的标准接口和服务发现机制仍然需要被进一步研究、规范。

当前也有很多智能设备相关的研究在考虑设备之间的互操作与服务发现机制，其中富于代表性的是~UPnP，该技术的无中心化工作方式更适用于能力和可靠性都受到局限的用户侧智能外设，然而，UPnP~并不考虑用户周边设备在随用户移动时的移动性特征，比如，大量的设备可能会因为用户的移动而加入到用户周围或是从用户周围消失，而这对于移动业务是至关重要的，这就需要我们加之以一定的研究。

\section{论文的主要工作}

本论文主要围绕着以下研究点展开，对移动环境下个人周边智能设备的协同工作与自组织中的部分问题进行一些研究:
\begin{itemize}
	\item \textbf{移动个人网络}的建模问题。移动个人网络由于异构性、个人周边环境的短距离特性和个人的移动性具有与一般的个人域网络~(PAN)~和移动自组织网络~(MANET)~所不同的特性，因此需要对其进行建模。
	%\item 设备的基于移动性的分组问题。个人周边设备一般附属于衣物、车辆、房间等不同的实体，这些设备的加入与退出移动个人网络是与这些实体的相对移动相关联的，如何自动配置设备、使之互相识别出这一分组是一个值得研究的问题。
	\item 移动个人网络中的服务发现问题。在上述模型和分组移动性研究的基础上，服务的发现的研究最终将服务于移动泛在业务。
	\item 移动个人网络中信息的分布式存储问题。针对上述移动性特征，研究如何将系统信息高效、可靠地存储于这一分布式系统之中，并尽量节省带宽资源和节点的存储资源。
\end{itemize}

在论文本课题之外，安全性相关的研究、多用户条件下的设备协同工作的研究都是下一步的研究范畴。本课题的研究工作基于国家~863~项目``基于移动代理技术的个人移动性研究''、``下一代信息网中统一业务终端技术''~和中德合作、科技部资助~``增强统一业务终端技术: 个人域网络和广域网的协同工作''~等项目。

本论文课题的成果不仅包括论文输出，截止~2007~年已经成功申请两项国家发明专利，其他专利正在申请过程中，同时，研究结果被实现于实验室的演示平台当中，其中的一些技术点进一步会实现为实验室内部服务，可以方便实验室老师、同学和工作人员的工作和生活。

\section{论文的结构}

本论文将以层层深入的方式，由表及里地阐述作者对于个人周边智能设备的协同与自组织机制的研究。从分布式智能终端系统~――~个人周边智能设备自组织而成的服务于用户的分布式系统的应用场景、概述和系统结构开始，介绍作者在此领域的已有研究，之后进一步深入，分析作为这一分布系统的底层基础设施的移动个人网络的行为特点，建立移动性模型，并在此基础上，给出移动个人网络的基本机制，服务发现与系统信息的分布存储。

第~\ref{ch:ust}~章详细论述了将个人周边的智能设备协同构成分布式终端的相关研究，提出了聚合重构分布式终端的概念，将重构的概念从被动适应网络连接的变化拓展到了主动协同用户周边智能设备，为用户提供恰当的服务。第~\ref{ch:ust}~章介绍了论文作者在此领域进行的前期架构研究，描述了聚合重构分布式终端的应用场景和基本概念，归纳了终端的工作状态和系统架构，阐述了作为分布式终端组成元素的各个设备的结构和抽象接口，并提出了分布式终端系统的有中心和无中心两类组织方式，是后面各章研究的基础。

在第~\ref{ch:mm_sd}~章之中，首先对分布式终端所在的自组织的网络结构进行了界定，称为~``移动个人网络''，并对移动个人网络的连通性和移动性特征，进行了归纳和分析，并在前人所研究的移动自组织网络的移动性模型的基础之上，结合移动个人网络的特定情况，建立了移动个人网络的组移动性模型。利用个人移动网络的移动性特征，第~\ref{ch:mm_sd}~章同时给出了基于组移动性的个人移动网络的服务发现机制。

第~\ref{ch:data}~章在前面各章的研究基础之上，对服务发现机制和系统组织管理的底层支撑基础设施~――~分布式的数据存储机制进行了研究，通过比较有中心存储、非结构化无中心存储和结构化无中心存储机制，给出了适应于移动个人网络的行为特征的无中心分布式存储机制，具有良好的可靠性、稳定性和可伸缩性，与已有结构化无中心存储方法相比，占用更少的带宽资源，并且更能适应移动个人网络中大量节点同时加入或退出的情况。

最后，第~\ref{ch:conclusion}~章总结了论文的研究工作，并对将来的下一步研究工作进行了展望。
